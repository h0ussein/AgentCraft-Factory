# Research: Improving the project for recruiters (LinkedIn / portfolio)

**Goal:** Make it easy for recruiters on LinkedIn to find the project, understand it, try it, and see your skills.  
**Use this doc when you’re ready to implement** — no code was changed; this is research only.

---

## 1. What recruiters see today

| Where | Current state |
|-------|----------------|
| **GitHub repo** | No root README. Main docs in `PROJECT_SUMMARY.md`. Recruiters often expect a README at repo root. |
| **Running the app** | Need to copy env, set `GOOGLE_API_KEY`; `MONGO_URI` and `ADMIN_PASSCODE` are only in `PROJECT_SUMMARY.md` and `render.yaml`, not in `.env.example`. |
| **Sharing link** | No meta description or Open Graph tags in `frontend/index.html`, so LinkedIn/social previews may show a generic or empty description. |
| **“Try it”** | No obvious live demo link; recruiters must clone and configure. |
| **Skills visibility** | Tech stack is in `PROJECT_SUMMARY.md` but not in a short, scannable form at repo root or in the app. |

---

## 2. Recommended improvements (to do later)

### High impact, low effort

- **Add a root README.md**  
  - One-line pitch, “What it does” (bullet list), tech stack table, quick start (clone → copy `.env.example` to `backend/.env` → set keys → install → build → start).  
  - Optional: “For LinkedIn / portfolio” — a short blurb they can paste when adding the project to LinkedIn.  
  - Leave a placeholder for “Live demo: [URL]” to fill after deployment.

- **Document env in `.env.example`**  
  - Add `MONGO_URI` and `ADMIN_PASSCODE` with short comments (required for agents list, history, admin).  
  - Mention that only `GOOGLE_API_KEY` is needed for minimal run; others for full features.

- **Deploy a live demo**  
  - Use existing `render.yaml`; set `GOOGLE_API_KEY`, `MONGO_URI`, `ADMIN_PASSCODE` in Render dashboard.  
  - Put the live URL in README and in LinkedIn (Featured / Projects).  
  - Recruiters can “try it” without cloning.

- **Add meta + Open Graph in `frontend/index.html`**  
  - `meta name="description"` and `og:title`, `og:description`, `og:type` so shared links (e.g. LinkedIn) show a proper preview.

- **Screenshots or short GIF in README**  
  - 1–2 screenshots (e.g. Chat + My Agents or Create tool modal) or a ~15s GIF: open app → create agent → create tool → chat.  
  - Gives instant visual of the app.

- **LinkedIn + GitHub profile**  
  - LinkedIn Featured: add project with title, live demo URL (and repo link), short description + tech stack.  
  - GitHub: set repo “About” (short description + topics e.g. `react`, `fastapi`, `mongodb`, `gemini`, `portfolio`), pin repo to profile.

### Medium impact, medium effort

- **“About this project” / “Tech” in the app**  
  - Small link in header or footer opening a modal: what the app does, bullet list of tech, link to repo (and live demo).  
  - Recruiters who try the app see your skills without leaving.

- **Friendlier first-run and errors**  
  - When there are no agents: e.g. “Or chat with the default agent on the Chat tab.”  
  - When API key missing or DB down: short message like “Set GOOGLE_API_KEY in backend/.env to enable chat” instead of generic error.

- **Optional: `og:image`**  
  - Add a 1200×630 image and `<meta property="og:image" content="...">` so shared links show a nice preview image.

### Higher effort, high impact

- **Tests + CI**  
  - A few backend tests (e.g. FastAPI `TestClient` for `/api/health`, `/api/agents`) and/or frontend smoke tests; GitHub Actions to run on push/PR.  
  - Shows quality and collaboration.

- **Short “Architecture” in README or PROJECT_SUMMARY**  
  - One paragraph or diagram: frontend → FastAPI → MongoDB + Gemini; tools generated by Gemini and run in sandbox.  
  - Shows you can explain design.

- **Docker or “Contributing”**  
  - Docker Compose to run app + MongoDB with one command, or a short “Contributing” section (how to run, how to add an endpoint/component).  
  - Makes the project feel production-ready.

- **LinkedIn article or post**  
  - Short post: “I built an app that generates Python tools from natural language — here’s the stack and what I learned.” Link to demo and repo.  
  - Increases visibility and shows communication.

## 3. Skills to highlight (for README / LinkedIn)

- **Frontend:** React 19, Vite 7, Tailwind CSS, mobile-first UI.  
- **Backend:** Python, FastAPI, REST API design, Pydantic.  
- **Data:** MongoDB (PyMongo), session/agent modeling.  
- **AI/LLM:** Google Gemini 2.5 Flash, Agno agent framework, natural-language → code (tool generation).  
- **Security:** Tool code safety review, sandboxed execution (RestrictedPython), admin passcode.  
- **DevOps / deployment:** Single-server (FastAPI serves API + SPA), Render-ready config.
